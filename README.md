Лабораторная работа №2-3 
Тема: Объектно-ориентированное моделирование системы обработки запросов и мониторинга её 
характеристик на C# 
1. Цели работы 
1. Закрепить навыки объектно-ориентированного проектирования на языке C# (классы, 
интерфейсы, наследование, полиморфизм, инкапсуляция, композиция). 
2. Отработать применение базовых принципов системного проектирования: наблюдаемость, 
работа с задержками, устойчивость системы к отказам. 
3. Научиться строить расширяемую архитектуру (интерфейсы, паттерны проектирования) для 
моделирования поведения сервисов и обработки запросов. 
 
Этап 1. Общее задание 

1.1. Постановка задачи 

Необходимо разработать консольное приложение на C#, которое моделирует работу упрощённой 
распределённой системы обработки запросов. Система состоит из одного или нескольких сервисов, к 
которым поступают запросы. Для каждого запроса моделируются: 
• время обработки (задержка); 
• вероятность отказа; 
• итоговое состояние (успех/ошибка). 
Приложение должно вести простейшую наблюдаемость: 
• сбор метрик по каждому сервису; 
• вычисление агрегированных показателей (средняя задержка, максимальная задержка, процент 
ошибок); 
• оценка «здоровья» (состояния) сервиса на основе текущих метрик. 
Результаты моделирования необходимо выводить в консоль в структурированном виде.

Этап 2. Индивидуальные задания 

Вариант 11. Экспорт метрик в JSON 

К общему заданию добавить: 

1. Реализовать экспорт текущих метрик всех сервисов в файл в формате JSON. 
2. Структура JSON должна быть разумно организована (например, список объектов по сервисам). 
3. Обеспечить возможность последующей загрузки метрик из файла и их отображения.
